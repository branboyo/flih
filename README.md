# Flih

A Chrome extension that captures tab audio, lets you edit it with a waveform editor and FX chain, and exports to WAV.

## Features

- **Tab audio capture** — records the active tab's audio stream via Chrome's `tabCapture` API, opened as a side panel so it stays out of the way
- **Live waveform** — real-time oscilloscope visualization during recording using an `AnalyserNode` FFT
- **Waveform editor** — drag trim handles to set in/out points; playback loops seamlessly within the region
- **FX chain** — stackable, reorderable effects applied non-destructively at preview time and baked in at export: Pitch Shift, Delay, Reverb, Distortion, EQ, Reverse
- **Export** — saves to `Downloads/Flih/` as WAV (hand-written encoder, no library dependency)
- **Recording library** — persists recordings in IndexedDB; browse and reload past sessions
- **Settings** — configurable download folder, preferred format, sample rate, and waveform zoom style

## Algorithm Details

### Pitch Shift

Implemented via [`rubberband-wasm`](https://github.com/mmomtchev/rubberband-wasm) — a WebAssembly build of the [Rubber Band Library](https://breakfastquay.com/rubberband/).

**Engine:** `RubberBandOptionEngineFiner` (RBME — Rubber Band Music Engine). This is RubberBand 3's high-quality offline engine. It avoids the phase-vocoder phasiness artifacts of the classic STFT approach by using a different spectral model internally.

**Signal flow:**
1. WASM module is lazy-loaded and compiled once (`WebAssembly.compile`), then cached for the lifetime of the extension.
2. Per-channel `Float32Array` buffers are written directly onto the WASM heap via `malloc` + `memWrite`.
3. A **study pass** (`rubberband_study`) lets RubberBand pre-analyse the full signal before processing — required for the offline engine to make globally optimal decisions.
4. A **process pass** (`rubberband_process`) runs the actual pitch shift. `rubberband_set_expected_input_duration` is called before the study pass so the engine can size its internal buffers correctly.
5. Output is retrieved in chunks by polling `rubberband_available` + `rubberband_retrieve`, then concatenated per-channel into a new `AudioBuffer` at the actual output length (which may differ from the input due to pitch-shift latency compensation).
6. All WASM heap allocations are freed in a `finally` block.

**Options:** `ProcessOffline | EngineFiner | ChannelsTogether` plus optional `FormantPreserved`. `ChannelsTogether` locks stereo channel phases together, preventing inter-channel drift.

**Controls:** ±12 semitones, ±100 cents fine-tune, formant preservation toggle.

### Distortion

Uses the Web Audio API `WaveShaperNode` with a hand-computed `tanh` saturation curve.

- **Curve shape:** `tanh(x * gain)` where `gain = 200^(drive/100)`. The exponential mapping means equal knob travel feels perceptually uniform across the full range (drive 0 → 1× gain ≈ clean; drive 50 → 14×; drive 100 → 200× ≈ hard fuzz).
- **Pre-gain baked into the curve** rather than a separate `GainNode`, so the waveshaper's built-in `4x` oversampling applies to the entire nonlinearity — this reduces aliasing from the high-frequency harmonics generated by hard clipping.
- **Post-clip tone shelf:** a `BiquadFilterNode` (highshelf, 2 kHz) sits after the clipping stage, matching the convention of analog pedal tone stacks. Negative values roll off highs (warm); positive values add presence.
- **Level compensation:** at high drive, `tanh` saturates near ±1 and the signal gets louder. The wet gain is scaled by `0.5^(drive/100)` to keep the wet path roughly level-matched with the dry path.
- **Dry/wet mix** applied with a simple parallel gain split.

---

### Reverb

Convolution reverb via `ConvolverNode` with a **synthetically generated impulse response**.

- IR is generated as exponentially decaying white noise: `sample[i] = rand() * exp(-t * (10 / (decay * duration)))`. This approximates the statistical energy decay of a real room without needing a recorded IR file.
- Room size maps to IR duration (0.5 s → 4 s); decay controls the time constant of the exponential envelope.
- `convolver.normalize = true` keeps the output level consistent regardless of IR length.
- After rendering, a **silent tail trimmer** scans backwards from the end of the output to find the last sample above −66 dBFS, then discards everything after that point plus a 100 ms pad. This prevents reverb tails from bloating the exported file.

---

### Delay

`OfflineAudioContext` doesn't support feedback loops (the graph must be acyclic), so the feedback is **manually unrolled**: up to 12 echo copies of the source buffer are scheduled at `t = delay × n`, each attenuated by `feedback^n`. The loop terminates early once the echo gain drops below 0.1%. The same silent tail trimmer clips the output after the last audible echo.

---

### EQ

Three-band parametric EQ using chained `BiquadFilterNode`s rendered offline:

| Band | Type | Frequency |
|------|------|-----------|
| Low | `lowshelf` | 200 Hz |
| Mid | `peaking` (Q = 1) | 1 kHz |
| High | `highshelf` | 5 kHz |

Each band's gain is in dB, matching the unit convention of every DAW.

---

### Seamless Loop Playback

WaveSurfer's `region.play()` schedules an internal pause at the region end, which creates a brief gap on each loop. Instead, the editor uses a `requestAnimationFrame` polling loop:

Each frame (~16 ms) while playing, `ws.getCurrentTime()` is compared against `region.end`. When it crosses the boundary, `ws.setTime(region.start)` repositions the playhead **without pausing** — `media.currentTime` is set directly, so the audio engine keeps running. The rAF loop is the sole loop authority; all `region.play()` calls are replaced with `ws.play()` (plain playback, no scheduled stop).

The `finish` event (fires when the clip ends naturally) acts as a fallback in case the rAF misses the boundary by one frame.

---

### WAV Encoder

Hand-written, zero-dependency encoder (`lib/encoder.ts`):

1. Constructs the 44-byte RIFF/WAVE/fmt/data header using `DataView` with little-endian writes.
2. Interleaves channel data sample-by-sample (L R L R …) and converts `Float32` → `Int16` with clamping: `int16 = sample < 0 ? sample * 0x8000 : sample * 0x7FFF`.
3. Returns a `Blob` with `type: "audio/wav"`.
